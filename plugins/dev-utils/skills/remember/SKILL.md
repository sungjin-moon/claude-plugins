프로젝트 컨벤션, 아키텍처 결정, 보완 사항, 베스트 프랙티스를 문서에 기록합니다.
이 스킬은 특정 프로젝트에 고정돼 있지 않다. **실행 시점에 프로젝트의 문서 구조를 먼저 파악하고** 그대로 따른다.

## 사용 시기

- 리뷰 중 발견한 패턴을 문서화하고 싶을 때
- 아키텍처 결정 사항을 기록하고 싶을 때
- 보완해야 할 점이나 개선 사항을 남기고 싶을 때
- 실수를 방지하기 위한 가이드를 추가하고 싶을 때
- 팀 컨벤션을 명시적으로 기록하고 싶을 때
- 반복되는 피드백을 프로젝트 가이드에 반영하고 싶을 때

## 예시

```bash
/remember                              # 대화 컨텍스트에서 자동 파악
/remember select 함수는 API에 두지 않음  # 직접 내용 지정
```

## Prompt

당신은 프로젝트 컨벤션과 베스트 프랙티스를 문서화하는 전문가입니다.

### 목표

사용자의 피드백, 아키텍처 결정, 보완 사항을 분석하여 적절한 프로젝트 문서에 기록합니다.

### 0. 문서 위치 파악 (항상 먼저)

기록 대상 문서는 일반적인 개념(아키텍처 문서 / CLAUDE.md / agents / memory)이지만, **실제 파일 경로는 프로젝트마다 다르므로 실행 시점에 확인한다.**

1. 프로젝트 루트에서 `.claude/conventions.json`이 있으면 읽는다:
   - `{{architectureDoc}}` = `architectureDoc` (기본 `ARCHITECTURE.md`)
2. 다음 문서들의 실제 존재 여부와 경로를 확인한다:
   - 아키텍처 문서: `{{architectureDoc}}` (프로젝트 루트, 기본 `ARCHITECTURE.md`)
   - `CLAUDE.md` (프로젝트 루트)
   - `.claude/agents/*.md` (있는 경우)
   - 세션 간 유지 메모: Claude Code의 프로젝트별 memory 파일(`MEMORY.md`). 경로는 환경마다 다르므로 **하드코딩하지 말고**, 현재 세션에서 로드된 memory 파일 경로를 사용하거나 없으면 사용자에게 확인한다.
3. 위 문서 중 일부가 없으면 있는 것만 대상으로 한다.

### 대상 문서 (개념별 용도)

| 문서 | 용도 | 예시 |
|------|------|------|
| **아키텍처 문서**(`{{architectureDoc}}`) | 아키텍처 원칙, 설계 가이드, 컨벤션 | API 패턴, 폴더 구조, 컴포넌트 규칙 |
| **CLAUDE.md** | 핵심 규칙 요약, 개발 프로세스 | 빌드 명령어, 앱 구조, 핵심 규칙 |
| **.claude/agents/*.md** | 에이전트별 작업 가이드 | API 빌더, 리뷰어, UI 빌더 규칙 |
| **memory 파일**(`MEMORY.md`) | 세션 간 유지되는 메모 | 아키텍처 결정, 사용자 선호, 프로젝트 노트 |

### 작업 순서

1. **컨텍스트 파악**
   - $ARGUMENTS가 있으면 그 내용을 기록 대상으로 사용
   - 없으면 최근 대화 내용 분석하여 기록할 내용 파악
   - 변경된 파일이 있다면 그 맥락 이해

2. **사용자에게 질문** ($ARGUMENTS가 없는 경우)

   ```
   어떤 내용을 기록할까요?

   예시:
   - "select 함수는 API 모듈이 아닌 페이지 mappers에 배치"
   - "Table header는 ReactNode 타입으로 변경됨"
   - "신규 앱은 공용 UI 패키지를 직접 import해야 함"
   ```

3. **문서 분석 (Task tool 사용)**

   Task tool로 `general-purpose` agent를 실행:

   ```typescript
   Task(
     subagent_type: "general-purpose",
     description: "문서 분석 및 위치 파악",
     prompt: `
       다음 내용을 프로젝트 문서에 추가하려고 합니다:
       "${사용자 입력}"

       작업:
       1. 대상 문서 읽기 (존재하는 것만):
          - 아키텍처 문서 (프로젝트 루트, 기본 ARCHITECTURE.md)
          - CLAUDE.md (프로젝트 루트)
          - .claude/agents/*.md
          - 세션 간 유지 memory 파일(MEMORY.md) — 경로는 현재 세션에 로드된 memory 파일 경로를 사용
       2. 내용 성격에 따라 적절한 문서 선택
       3. 해당 문서의 기존 내용 확인
       4. 중복/충돌 여부 체크
       5. 추가할 내용을 마크다운 형식으로 정리
       6. 위치와 내용을 제안 (여러 문서에 걸쳐 반영이 필요하면 모두 제안)

       제안 형식:
       - 파일: [파일명]
       - 섹션: [섹션명]
       - 추가 내용: [마크다운]
       - 이유: [왜 이 위치가 적절한지]
     `
   )
   ```

4. **제안 표시**

   Agent 결과를 사용자에게 보기 좋게 표시:

   ```markdown
   제안: [파일명] > [섹션명]

   추가할 내용:
   ```

   [마크다운 내용]

   ```

   이유: [설명]

   이 내용을 추가할까요?
   - yes: 그대로 추가
   - edit: 수정 후 추가
   - no: 취소
   ```

5. **사용자 응답에 따라 처리**
   - **yes**: Edit 또는 Write tool로 문서 업데이트
   - **edit**: 사용자가 수정할 수 있도록 내용 제시 -> 수정 후 반영
   - **no**: 취소

6. **완료 메시지**

   ```
   [파일명]에 내용이 추가되었습니다.
   [간단한 요약]
   ```

### 중요 규칙

1. **적절한 위치 선택 기준**

   | 내용 유형 | 대상 문서 | 섹션 |
   |-----------|-----------|------|
   | 폴더 구조/네이밍 | 아키텍처 문서 | 폴더 네이밍 컨벤션 |
   | 컴포넌트 사용법 | 아키텍처 문서 | 컴포넌트 재사용 가이드 |
   | API 패턴 | 아키텍처 문서 | API 설계 가이드 |
   | 핵심 규칙 변경 | CLAUDE.md | 핵심 규칙 |
   | 개발 프로세스/명령어 | CLAUDE.md | 핵심 명령어 |
   | 에이전트 동작 개선 | .claude/agents/[name].md | 해당 섹션 |
   | Skill 개선사항 | .claude/skills/[name]/SKILL.md | 해당 섹션 |
   | 아키텍처 결정/프로젝트 노트 | memory 파일(MEMORY.md) | Architecture Decisions / Project Notes |
   | 새로운 카테고리 | 적절한 문서에 새 섹션 생성 | - |

   **여러 문서에 동시 반영이 필요한 경우**: 예를 들어 API 패턴 변경은 아키텍처 문서(상세 가이드) + CLAUDE.md(핵심 규칙 요약) + 관련 에이전트 가이드 + memory 파일(결정 기록) 모두에 반영할 수 있음.

2. **중복 방지**
   - 기존에 유사한 내용이 있으면 수정 제안
   - 새로운 내용이면 추가 제안

3. **명확성**
   - 가능하면 좋은 예와 나쁜 예를 포함
   - 코드 블록 사용
   - 이유 설명

4. **간결성**
   - 핵심만 기록
   - 너무 장황하지 않게
   - 실용적인 예시 위주

### 주의사항

- 사용자의 의도를 정확히 파악
- 기존 문서 구조와 톤을 유지
- 너무 세세한 내용은 피하고 재사용 가능한 패턴 위주로
- 프로젝트 특성(모노레포, 신규/레거시 구분 등)을 고려
- memory 파일(MEMORY.md)은 200줄 이내로 유지 (초과 시 별도 파일로 분리)
